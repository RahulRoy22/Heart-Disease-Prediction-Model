# GitHub Push Guide

## Step-by-Step Instructions to Push Your Project to GitHub

### Step 1: Fix the Protobuf Conflict First

Open your terminal/command prompt and run:

```bash
cd "c:\Work\prediction model\Heart-Disease-Prediction-Model"
pip uninstall protobuf -y
pip install protobuf==4.25.3
```

Then reinstall requirements:
```bash
pip install -r requirements.txt
```

### Step 2: Initialize Git Repository (if not already done)

```bash
cd "c:\Work\prediction model\Heart-Disease-Prediction-Model"
git init
```

### Step 3: Add Files to Git

```bash
# Add all files (respecting .gitignore)
git add .

# Check what will be committed
git status
```

You should see:
- ✅ Heart_Disease_Prediction_Model_Upgraded.ipynb
- ✅ data.csv
- ✅ requirements.txt
- ✅ README.md
- ✅ .gitignore
- ❌ NOT: .pkl, .h5 files (ignored)
- ❌ NOT: UPGRADE_GUIDE.md, BEFORE_AFTER_COMPARISON.md (ignored)

### Step 4: Commit Changes

```bash
git commit -m "Advanced ML upgrade: Added ensemble stacking, deep learning, and full pipeline"
```

### Step 5: Create GitHub Repository

1. Go to https://github.com
2. Click "+" → "New repository"
3. Name it: `Heart-Disease-Prediction-Model`
4. Description: "Advanced ML system for heart disease prediction with ensemble methods and deep learning"
5. Choose: Public or Private
6. **DO NOT** initialize with README (we already have one)
7. Click "Create repository"

### Step 6: Connect Local to GitHub

Copy the commands from GitHub (they'll look like this):

```bash
git remote add origin https://github.com/YOUR_USERNAME/Heart-Disease-Prediction-Model.git
git branch -M main
git push -u origin main
```

Replace `YOUR_USERNAME` with your actual GitHub username.

### Step 7: Verify Upload

Go to your GitHub repository URL and verify:
- ✅ README.md displays nicely
- ✅ Notebook file is present
- ✅ data.csv is present
- ✅ requirements.txt is present
- ✅ .gitignore is working (no .pkl or .h5 files)

## Alternative: Using GitHub Desktop

4. Choose: `c:\Work\prediction model\Heart-Disease-Prediction-Model`

## Troubleshooting

### Issue: "fatal: not a git repository"
**Solution:**
```bash
git init
```

### Issue: "remote origin already exists"
**Solution:**
```bash
git remote remove origin
git remote add origin https://github.com/YOUR_USERNAME/Heart-Disease-Prediction-Model.git
```

### Issue: Large files error
**Solution:** Check .gitignore is working:
```bash
git status
```
If you see .pkl or .h5 files, they shouldn't be there. Remove them:
```bash
git rm --cached *.pkl
git rm --cached *.h5
```

### Issue: Protobuf conflicts
**Solution:**
```bash
pip install --upgrade protobuf==4.25.3 --force-reinstall
```

## After Pushing

### Update README with Your Info

Edit README.md and replace:
- `yourusername` with your GitHub username
- `your-email@example.com` with your email
- Add your name in the Author section

Then commit and push:
```bash
git add README.md
git commit -m "Updated README with personal info"
git push
```

### Add a License (Optional)

```bash
# Create LICENSE file
echo "MIT License" > LICENSE
git add LICENSE
git commit -m "Added MIT license"
git push
```

## Quick Reference Commands

```bash
# Check status
git status

# Add all changes
git add .

# Commit changes
git commit -m "Your message here"

# Push to GitHub
git push

# Pull latest changes
git pull

# View commit history
git log --oneline
```
